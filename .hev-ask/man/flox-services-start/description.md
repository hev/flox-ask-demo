Explains that the command starts specified services, warns (but succeeds) for services already running, errors and starts nothing if a named service doesn't exist, starts from an ephemeral activation using the latest build when none are running (which may differ from your shell's environment, so reactivate after manifest edits), and that a remote environment has only one set of running services regardless of concurrent activations.
