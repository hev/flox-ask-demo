Describes the publishing process: which build name to specify when multiple builds exist, that Flox clones the repository to a temporary location and runs a clean build to ensure all required files are tracked, that package metadata is sent to Flox servers, that the package and its dependencies are uploaded to the catalog store (Flox-provided by default or your own), and that the package goes to your user catalog by default or a specified catalog.
