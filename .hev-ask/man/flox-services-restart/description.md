Explains that the command restarts specified services, that with no arguments it stops all running services and starts new ones from the latest build, that named services restart using the config the running services started with, that restarting all uses an ephemeral activation that may differ from your shell's environment (so reactivate after manifest edits), and that a nonexistent service errors.
