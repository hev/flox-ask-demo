Explains that uploading a package requires signing it and installing a published package requires trusting the public key it was signed with. By default packages are signed with a key bundled in the Flox installer so user-published packages install out of the box, but providing your own Catalog Store requires creating signing keys, distributing the private key to publishers and the public key to installers, and having installers trust the public key.
