Demonstrates layering a project-specific environment on top of the default one by cloning an example project, initializing an environment, installing the project's dependency, and activating it. Both the project dependency and the default tools are then available; you can layer arbitrarily many environments, and when two contain the same package the last activated wins. A command lists currently active environments.
