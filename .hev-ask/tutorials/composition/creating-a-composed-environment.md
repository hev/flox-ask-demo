Explains composition, where a composing environment includes one or more included environments treated as dependencies. The included environment's manifest is merged into the composing one, project-specific packages can be added directly to the composing environment, and when manifests conflict the priority order decides, with the composing environment always winning; overridden fields are reported during merging.
