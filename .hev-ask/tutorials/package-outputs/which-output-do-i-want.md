Notes there are no hard rules for which outputs exist or when a package is split, an idiosyncrasy inherited from Nixpkgs, but lists common conventional output names and their typical contents (executables, manual pages, libraries, development headers, debug symbols, and the main output). It flags that some packages deviate, for example placing a dynamic library in the main output rather than a library output.
