Concludes that jobs targeted to the configured runner execute with Imageless Kubernetes, so using the same development environment lets you test with identical packages regardless of system or architecture. Highlights that because CI doesn't rely on a container image, updates are instant: pushing the environment makes jobs pick up the changes automatically.
