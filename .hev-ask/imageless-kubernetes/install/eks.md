Introduces installing Imageless Kubernetes on a managed cloud Kubernetes service, recommending a dedicated node group and walking through configuring it with both an infrastructure-as-code tool and the provider's cluster CLI. Lists what each node needs (install Flox, install and register the containerd runtime shim with containerd and Kubernetes), notes most of this happens during node bootstrapping, and points to example configurations for creating new clusters.
