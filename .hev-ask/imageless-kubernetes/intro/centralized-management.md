Explains that each pod specifies a command and the Flox environment to run it in, and the environment must be one pushed to FloxHub, which is where centralized management happens. Once pushed, FloxHub becomes the source of truth and provides the list of installed packages and versions, environment variables, startup scripts, and an audit trail of changes and who made them.
