Explains the implicit default package group named toplevel into which any package installed without specifying a group is placed, all resolving against the same pinned commit. Shows how to assign packages to a named group, notes that different groups may or may not resolve to the same commit (the resolver shares a commit when possible and diverges only when required), and observes that groups can also be used purely to organize the manifest.
