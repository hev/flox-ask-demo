Defines a Flox Catalog as a collection of package artifacts and metadata consumable by environments, searchable, showable, and installable via the corresponding commands. Distinguishes the Flox-populated Base Catalog (drawn from the Nix community's nixpkgs) from custom catalogs maintained by users and organizations via publishing, notes custom catalogs can be public or private, and defines a package as a bundle of programs and data declared in the environment manifest.
