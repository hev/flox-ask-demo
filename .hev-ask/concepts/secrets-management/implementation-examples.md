Provides implementation examples of the just-in-time secrets pattern across multiple secret stores (a macOS keychain, a password manager, a secrets vault, a cloud secrets manager, and cross-platform setups), each retrieving a secret in the on-activate hook. Notes which CLI and active session each requires, how to add that CLI to the environment, the macOS keychain access dialog behavior in interactive versus non-interactive contexts, and when a per-platform conditional is needed.
